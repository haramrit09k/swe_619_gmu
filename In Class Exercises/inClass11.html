SWE 619 In Class Exercise Number 11

This is a JUnit theory exercise.

  1. Recall the equals() and hashCode() discussion in Bloch. Write a JUnit theory that encodes the consistency property between equals() and hashCode().


  2. Provide some "data points" for the theory. Given this set of data points:
    • How many combinations are considered by the theory?
    • How many combinations make it past the preconditions of the theory?
    • How many combinations make it to the postcondition of the theory?



  3. Build a toy example that violates the theory. Fix the toy example so that the theory is no longer violated.


  4. Repeat the exercise for the symmetric and transitive properties for equals().


  5. What happens to these theories/data points when favoring composition over inheritance?


  6. Consider the Comparable interface: what properties should be checked with theories?
